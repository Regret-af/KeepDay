# 统计规则说明

## 1. 统计指标

KeepDay MVP 有四个核心统计指标。

| 指标 | 定义 | 展示位置 |
|---|---|---|
| 持续天数 | 从习惯创建日到今天的自然天数 | 详情页 |
| 累计打卡次数 | 当前习惯所有 CheckInRecord 数量 | 首页、详情页 |
| 最佳连续天数 | 历史最长未破坏连续链路天数 | 详情页 |
| 当前连续天数 | 当前正在延续的连续天数 | 首页、详情页 |

## 2. 权威数据来源

统计计算以以下数据为准：

- Habit.createdAt。
- Habit.gracePerWeek。
- CheckInRecord。
- HabitPausePeriod。
- 设备本地今天日期。

Habit 表中的统计字段是缓存，可重算。

## 3. 持续天数

```text
持续天数 = 今天日期 - 创建日期 + 1
```

规则：

- 不因未打卡归零。
- 包含宽限日。
- 包含暂停期间。
- 删除习惯不展示。

## 4. 累计打卡次数

```text
累计打卡次数 = CheckInRecord 总数
```

规则：

- 宽限日不生成记录，不计入累计。
- 暂停期间补卡会生成记录，计入累计。
- 取消打卡后数量减少。

## 5. 日期状态

计算月历和热力图时，日期状态统一使用：

```text
checked
checkedDuringPause
grace
missed
paused
future
beforeCreated
```

优先级：

```text
future / beforeCreated
→ checkedDuringPause
→ checked
→ paused
→ grace
→ missed
```

说明：

- 暂停期内有打卡记录，显示为 checkedDuringPause。
- 暂停期内无打卡记录，显示为 paused。
- 非暂停期未打卡且在宽限范围内，显示为 grace。
- 非暂停期未打卡且超出宽限，显示为 missed。

## 6. 宽限期规则

宽限期按自然周计算：

```text
周一 00:00 到周日 23:59
```

规则：

- 每个习惯独立设置 gracePerWeek。
- 可选值为 0 / 1 / 2。
- 宽限次数不跨周累计。
- 创建当周从创建日开始计算。
- 暂停日期不参与当周宽限消耗。
- 未打卡日期按日期从早到晚消耗宽限。
- 超出宽限次数的未打卡日为 missed。
- 宽限日不生成 CheckInRecord。
- 宽限日不计入累计打卡。
- 历史宽限日计入连续链路。
- 今天如果是宽限日且未打卡，不计入 currentStreak。

## 7. 最佳连续天数

最佳连续天数表示历史最长连续链路。

连续链路包含：

- 已打卡日期。
- 历史宽限日期。

连续链路跳过：

- 暂停日期。

连续链路中断：

- 非暂停日期。
- 非宽限日期。
- 未打卡日期。

暂停期间补卡：

- 计入累计打卡。
- 不计入最佳连续链路。

## 8. 当前连续天数

当前连续天数从今天向过去回溯。

规则：

- 今天已打卡：包含今天。
- 今天未打卡但处于宽限中：最多计算到昨天，不包含今天。
- 今天未打卡且不是宽限日：当前连续为 0。
- 今天处于暂停中：计算到暂停前最后一个有效节点。
- 暂停日期跳过，不增加，也不中断。

## 9. 计算建议

建议实现：

```text
StreakCalculator.calculate(
  habit,
  records,
  pausePeriods,
  today,
)
```

返回：

```text
HabitStats
Map<LocalDate, DateStatus>
```

计算范围：

- 从 habit.createdAt 的日期开始。
- 到 today 结束。

性能：

- 自用 MVP 数据量较小，可全量重算单个 habit。
- 每次写操作后只重算受影响 habit。
- 恢复后重算全部未删除 habit。

## 10. 必测样例

- 宽限期为 0，中断一天后 currentStreak 为 0。
- 宽限期为 1，周内第一个未打卡日为 grace。
- 宽限期为 1，周内第二个未打卡日为 missed。
- 今天是 grace 且未打卡，不计入 currentStreak。
- 历史 grace 计入 bestStreak。
- 暂停期间不消耗宽限。
- 暂停期间补卡不计入连续。
- 恢复后连续从暂停前继续延伸。
