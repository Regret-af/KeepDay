# 状态流转说明

## 1. Habit 状态

Habit 主状态：

```text
normal
paused
deleted
```

其中 deleted 不是 status 字段值，而是：

```text
deletedAt != null
```

## 2. Habit 状态流转

```text
创建
→ normal

normal
→ paused
→ normal

normal
→ deleted

paused
→ deleted
```

规则：

- deleted 状态不可在 App 内恢复。
- deleted 不参与展示、统计、通知、导出。
- paused 可编辑、可恢复、可删除。

## 3. 今日状态

首页今日状态：

```text
notChecked
checked
resting
paused
```

判断规则：

```text
如果 habit.status = paused → paused
否则如果今天存在 CheckInRecord → checked
否则如果今天是宽限日 → resting
否则 → notChecked
```

首页正常列表不展示 paused，paused 进入折叠区。

## 4. 日期状态流转

单个日期可能状态：

```text
beforeCreated
future
checked
checkedDuringPause
grace
missed
paused
```

常见流转：

```text
missed → checked        补卡
grace → checked         宽限日补卡
paused → checkedDuringPause 暂停期补卡
checked → missed/grace  取消打卡后重算
checkedDuringPause → paused 取消暂停期打卡
```

注意：

- grace 不是持久化状态，会随补卡、取消打卡、宽限期修改而重算。
- paused 由 HabitPausePeriod 决定。

## 5. 打卡记录状态

CheckInRecord 本身不做软删除。

```text
不存在
→ 创建记录
→ 存在
→ 删除记录
→ 不存在
```

创建来源：

```text
today
makeup
```

规则：

- 今天打卡 source = today。
- 历史补卡 source = makeup。
- 今天在详情页补打也使用 today。

## 6. 暂停区间状态

暂停区间：

```text
active: endDate = null
closed: endDate != null
```

流转：

```text
暂停习惯
→ 创建 active pause period

恢复习惯
→ active pause period 填充 endDate
→ closed
```

约束：

- 一个 habit 同一时间最多一个 active pause period。
- habit.status = paused 时必须存在 active pause period。
- habit.status = normal 时不应存在 active pause period。

## 7. 通知状态

NotificationLog 状态：

```text
sent
skipped
failed
```

规则：

- 每个 habit 每天最多一条通知记录。
- 通知发送失败不影响打卡状态。
- 权限关闭时可记录 skipped 或不记录，MVP 建议记录 skipped，便于排查。

## 8. 备份恢复状态

恢复流程状态：

```text
idle
→ pickingFile
→ validating
→ waitingConfirm
→ restoring
→ success
→ failed
```

失败场景：

- JSON 格式错误。
- version 缺失。
- version 过高。
- 必填字段缺失。
- ID 引用不完整。
- 唯一约束冲突。
- 写入事务失败。

失败要求：

```text
当前本地数据未被修改
```

## 9. 页面加载状态

建议统一：

```text
loading
data
empty
error
```

首页空状态：

- 无任何未删除习惯。

全部暂停状态：

- 有未删除习惯。
- 全部 status = paused。

详情页无记录：

- 当前 habit 没有 CheckInRecord。
