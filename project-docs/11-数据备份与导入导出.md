# 数据备份与导入导出

## 1. 功能目标

KeepDay MVP 不做账号和云同步，因此必须提供本地 JSON 备份与覆盖恢复能力。

目标：

- 用户可以导出自己的本地数据。
- 用户可以通过 JSON 文件恢复数据。
- 恢复失败时当前数据不被修改。
- 备份格式具备版本兼容能力。

## 2. 备份格式

MVP 使用明文 JSON。

顶层结构：

```json
{
  "version": "1.0.0",
  "exportedAt": "2026-05-27T00:00:00+08:00",
  "habits": [],
  "checkInRecords": [],
  "notificationLogs": [],
  "habitPausePeriods": []
}
```

## 3. 导出范围

导出包含：

- deletedAt 为 null 的 Habit。
- 未删除 Habit 关联的 CheckInRecord。
- 未删除 Habit 关联的 NotificationLog。
- 未删除 Habit 关联的 HabitPausePeriod。

导出不包含：

- 已软删除 Habit。
- 已软删除 Habit 的关联数据。
- App 内部临时状态。
- 系统通知计划。
- 用户设备权限状态。

## 4. 导出流程

```text
用户点击导出
→ 查询未删除 Habit
→ 查询关联数据
→ 生成 BackupPayload
→ JSON 序列化
→ 生成文件名
→ 调起系统保存/分享
→ 成功提示
```

推荐文件名：

```text
keepday-backup-YYYYMMDD-HHmm.json
```

## 5. 恢复模式

MVP 仅支持覆盖恢复。

不支持：

- 合并恢复。
- 选择性恢复。
- CSV 恢复。
- 云端恢复。

覆盖恢复文案必须明确说明：

```text
恢复备份将覆盖当前本地数据。
当前 App 内已有习惯和打卡记录会被替换为备份文件中的内容。
```

## 6. 恢复校验

写入数据库前必须完成全部校验。

### 6.1 JSON 格式校验

- 必须是合法 JSON。
- 顶层必须是对象。
- 必须存在 version。
- 必须存在 habits。
- 必须存在 checkInRecords。
- 必须存在 notificationLogs。
- 必须存在 habitPausePeriods。

### 6.2 版本校验

规则：

- 同版本必须支持恢复。
- 低小版本尽量兼容。
- 高版本禁止恢复。
- 不支持版本禁止恢复。

### 6.3 字段校验

Habit：

- id 不为空。
- name 合法。
- color 不为空。
- gracePerWeek 只能是 0 / 1 / 2。
- status 只能是 normal / paused。
- reminderEnabled 为 true 时 reminderTime 必须存在。

CheckInRecord：

- id 不为空。
- habitId 必须存在于 habits。
- checkInDate 合法。
- source 只能是 today / makeup。
- habitId + checkInDate 不重复。

HabitPausePeriod：

- habitId 必须存在于 habits。
- startDate 合法。
- endDate 如果存在，不能早于 startDate。

NotificationLog：

- habitId 必须存在于 habits。
- notifyDate 合法。
- notifyTime 合法。
- status 合法。

## 7. 恢复事务

恢复必须事务化。

流程：

```text
开始事务
→ 清空 notification_logs
→ 清空 habit_pause_periods
→ 清空 check_in_records
→ 清空 habits
→ 写入 habits
→ 写入 check_in_records
→ 写入 habit_pause_periods
→ 写入 notification_logs
→ 重算所有 habit 统计缓存
→ 提交事务
```

失败时：

```text
rollback
→ 当前数据保持原样
→ 展示恢复失败提示
```

## 8. 恢复后的处理

恢复成功后：

- 刷新 Riverpod 页面状态。
- 取消当前所有已计划通知。
- 按恢复后的 habit 重新安排未来通知。
- NotificationLog 仅作为历史记录，不触发过去通知。
- 返回首页。
- 提示“恢复完成”。

## 9. 备份兼容策略

建议维护：

```text
BackupSchemaVersion.current = 1.0.0
```

新增字段：

- 提供默认值。
- 旧版本恢复时自动补齐。

删除字段：

- 至少保留一段兼容解析逻辑。

高版本：

- 不尝试降级恢复。
- 明确提示用户升级 App。

## 10. 安全说明

MVP 备份为明文 JSON。

关于页和导出页应提示：

- 备份文件可能包含习惯名称和打卡记录。
- 请用户自行妥善保管。
- 后续版本可考虑加密备份。
