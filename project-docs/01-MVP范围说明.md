# KeepDay MVP 范围说明

## 1. 文档目的

本文定义 KeepDay 自用 MVP 的开发范围、阶段边界和不做事项，用于约束 Flutter 首版开发。

技术路线已确定为：

```text
Flutter + Riverpod + Drift(SQLite) + 本地通知 + JSON 文件导入导出
```

## 2. 产品定位

KeepDay 是一款极简、本地优先的习惯打卡 App。

核心价值：

- 首页快速查看今日习惯。
- 首页一键完成今日打卡。
- 支持补卡、取消打卡、宽限期和暂停。
- 使用本地 SQLite 保存数据。
- 支持本地 JSON 导出与覆盖恢复。
- 不依赖账号、云同步、社交或积分体系。

## 3. MVP 必须完成

### 3.1 习惯管理

- 创建习惯。
- 编辑习惯。
- 软删除习惯。
- 暂停习惯。
- 恢复习惯。
- 设置颜色标签。
- 设置 emoji 图标。
- 设置每周宽限期：0 / 1 / 2 天。
- 设置每日提醒开关和提醒时间。

### 3.2 首页

- 展示 App 名称和今日日期。
- 展示今日完成进度。
- 展示今日习惯列表。
- 支持今日打卡。
- 支持取消今日打卡。
- 展示宽限中状态。
- 展示通知权限提示。
- 展示已暂停习惯折叠区。
- 提供创建习惯入口。
- 提供设置页入口。

### 3.3 习惯详情

- 展示习惯基础信息。
- 展示持续天数。
- 展示累计打卡次数。
- 展示最佳连续天数。
- 展示当前连续天数。
- 展示月历视图。
- 展示热力图。
- 支持历史补卡。
- 支持取消历史打卡。
- 支持编辑、暂停、恢复、删除。

### 3.4 本地通知

- 每个习惯最多一个每日提醒时间。
- 到提醒时间时，如果当天未打卡且习惯未暂停，则发送提醒。
- 通知权限未开启时，首页持续提示。
- 删除、暂停、恢复、编辑提醒时间后重新处理通知计划。
- 使用 NotificationLog 记录提醒结果。

### 3.5 数据备份与恢复

- 支持 JSON 导出。
- 只导出未删除习惯及其关联数据。
- 支持 JSON 覆盖恢复。
- 恢复前校验文件结构和版本。
- 高版本备份禁止恢复。
- 恢复必须事务化。
- 恢复后重算统计缓存并重建通知计划。

### 3.6 空状态和错误状态

- 首页无习惯。
- 全部习惯暂停。
- 详情页无记录。
- 通知权限未开启。
- 备份文件错误。
- 备份版本不支持。
- 恢复失败。

## 4. MVP 可延后

- 手动排序。
- CSV 导出和恢复。
- 完成率、周趋势、月趋势等复杂统计。
- 更精致动画。
- 应用锁、生物识别。
- 加密备份。
- 云同步。
- 登录注册。

## 5. MVP 明确不做

- 社交分享。
- 好友系统。
- 排行榜。
- 积分系统。
- 付费功能。
- AI 功能。
- 一天多次打卡。
- 多个提醒时间。
- 自定义打卡频率。
- 合并恢复。
- App 内回收站。
- 历史日期列表。

## 7. MVP 验收标准

- 用户 1 分钟内可以创建第一个习惯。
- 用户打开首页后可以一键完成今日打卡。
- 打卡、补卡、取消打卡后统计即时正确。
- 宽限期和暂停期不会造成统计歧义。
- 删除习惯后前台不可见，且不参与导出。
- 通知权限关闭时有明确提示。
- JSON 恢复失败时当前数据不被修改。
