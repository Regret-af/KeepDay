# 业务规则说明

## 1. 习惯名称规则

保存前：

- 自动 trim。
- trim 后不能为空。
- 不允许换行。
- 最多 30 个字符。
- 允许中文、英文、数字、符号和 emoji。

同名规则：

- 允许同名。
- 创建同名习惯时必须提示确认。
- 检测范围包括 normal 和 paused。
- 不包括 deletedAt 不为空的习惯。

## 2. 创建习惯

默认值：

```text
status = normal
deletedAt = null
gracePerWeek = 0
reminderEnabled = false
sortOrder = 当前最大值 + 1
totalCheckInCount = 0
bestStreak = 0
currentStreak = 0
```

创建成功后：

- 返回首页。
- 新习惯参与今日列表。
- 如开启提醒，则尝试安排本地通知。

## 3. 编辑习惯

允许编辑：

- 名称。
- 颜色。
- emoji。
- 宽限期。
- 提醒开关。
- 提醒时间。

规则：

- 编辑不影响历史打卡记录。
- 已暂停习惯仍允许编辑。
- 修改宽限期后，按当前设置重算统计和日期状态。
- 修改提醒设置后，重新安排通知。

## 4. 删除习惯

删除为软删除。

规则：

- 删除前必须二次确认。
- 设置 deletedAt。
- 前台不再展示。
- 不参与统计。
- 不参与通知。
- 不参与 JSON 导出。
- 取消该习惯后续通知计划。
- 关联记录保留。

## 5. 暂停习惯

暂停后：

- habit.status = paused。
- 新增 HabitPausePeriod。
- 暂停习惯不参与首页正常列表。
- 暂停习惯不参与今日完成进度。
- 暂停习惯不发送提醒。
- 暂停期间允许补卡。
- 暂停期间补卡计入累计打卡次数。
- 暂停期间日期不计入连续链路，也不中断连续链路。
- 暂停期间不消耗宽限次数。

## 6. 恢复习惯

恢复后：

- habit.status = normal。
- 填充当前 HabitPausePeriod.endDate。
- 从恢复当天开始参与首页列表。
- 从恢复当天开始参与通知。
- 连续链路从暂停前最后有效节点继续延伸。

## 7. 今日打卡

规则：

- 同一习惯同一日期只能存在一条 CheckInRecord。
- 今日打卡 source = today。
- 如果今天原本处于宽限中，打卡后状态变为已完成。
- 打卡后当天不再提醒该习惯。
- 打卡后重算统计缓存和首页状态。

## 8. 取消今日打卡

规则：

- 点击已完成按钮后弹出确认。
- 确认后删除今日 CheckInRecord。
- 取消后重算统计缓存。
- 如果今日通知已发送，不补发。
- 如果今日通知未发送，后续是否提醒取决于提醒时间是否还未到。

## 9. 补卡

规则：

- 只能补过去日期和今天。
- 创建日期之前不可补卡。
- 未来日期不可补卡。
- 每天最多一条记录。
- 历史日期 source = makeup。
- 今天在详情页打卡仍 source = today。
- 暂停期间允许补卡。
- 暂停期间补卡计入累计打卡次数，但不计入连续链路。
- 补卡不触发历史通知。

## 10. 取消历史打卡

规则：

- 取消前必须二次确认。
- 确认后删除对应 CheckInRecord。
- 取消后重算累计、最佳连续、当前连续、月历、热力图。

## 11. 首页完成进度

```text
今日已完成数量 = 今日已打卡且未暂停的习惯数量
今日应完成数量 = 当前未暂停习惯数量 - 今日处于宽限中且未打卡的习惯数量
休息中数量 = 今日未打卡但处于宽限中的未暂停习惯数量
```

优先级：

```text
已打卡 > 宽限中 > 未打卡
```

暂停习惯不计入首页进度。

## 12. 通知规则

发送条件：

- habit.deletedAt = null。
- habit.status = normal。
- habit.reminderEnabled = true。
- reminderTime 不为空。
- 今日未打卡。
- 今日不是暂停日。
- 今日尚未发送过通知。
- 系统通知权限可用。

通知只作为提醒，不改变打卡状态。

## 13. 备份恢复规则

导出：

- 只导出未删除 Habit。
- 只导出未删除 Habit 关联数据。
- 明文 JSON。

恢复：

- 仅支持 JSON。
- 覆盖恢复。
- 不支持合并。
- 必须事务化。
- 高版本备份禁止恢复。
- 恢复失败时当前数据不变。
