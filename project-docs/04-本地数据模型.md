# 本地数据模型

## 1. 模型总览

KeepDay MVP 包含四类核心业务数据：

```text
Habit
CheckInRecord
HabitPausePeriod
NotificationLog
```

关系：

```text
Habit 1 - N CheckInRecord
Habit 1 - N HabitPausePeriod
Habit 1 - N NotificationLog
```

## 2. Habit

习惯主体数据。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | String | 是 | 习惯唯一 ID |
| name | String | 是 | 习惯名称，最多 30 个字符 |
| icon | String? | 否 | emoji 图标 |
| color | String | 是 | 颜色，Hex 字符串 |
| reminderEnabled | bool | 是 | 是否开启提醒 |
| reminderTime | String? | 条件必填 | HH:mm |
| gracePerWeek | int | 是 | 每周宽限天数，0/1/2 |
| status | HabitStatus | 是 | normal / paused |
| sortOrder | int | 是 | 排序值，MVP 可默认 createdAt 顺序 |
| totalCheckInCount | int | 是 | 累计打卡缓存 |
| bestStreak | int | 是 | 最佳连续缓存 |
| currentStreak | int | 是 | 当前连续缓存 |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |
| deletedAt | DateTime? | 否 | 软删除时间 |

枚举：

```text
HabitStatus.normal
HabitStatus.paused
```

## 3. CheckInRecord

打卡记录。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | String | 是 | 记录唯一 ID |
| habitId | String | 是 | 关联 Habit |
| checkInDate | LocalDate | 是 | 打卡日期，YYYY-MM-DD |
| source | CheckInSource | 是 | today / makeup |
| createdAt | DateTime | 是 | 创建时间 |

枚举：

```text
CheckInSource.today
CheckInSource.makeup
```

约束：

```text
habitId + checkInDate 唯一
```

## 4. HabitPausePeriod

暂停区间。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | String | 是 | 暂停区间 ID |
| habitId | String | 是 | 关联 Habit |
| startDate | LocalDate | 是 | 暂停开始日期 |
| endDate | LocalDate? | 否 | 暂停结束日期，null 表示仍在暂停中 |
| createdAt | DateTime | 是 | 创建时间 |

规则：

- 暂停时新增一条记录。
- 恢复时填充当前未结束区间的 endDate。
- 一个习惯可有多个暂停区间。
- 判断某日是否暂停必须以该表为准。

## 5. NotificationLog

通知记录。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | String | 是 | 通知记录 ID |
| habitId | String | 是 | 关联 Habit |
| notifyDate | LocalDate | 是 | 通知日期 |
| notifyTime | String | 是 | HH:mm |
| status | NotificationStatus | 是 | sent / skipped / failed |
| createdAt | DateTime | 是 | 创建时间 |

枚举：

```text
NotificationStatus.sent
NotificationStatus.skipped
NotificationStatus.failed
```

建议约束：

```text
habitId + notifyDate 唯一
```

## 6. 派生模型

### 6.1 HabitStats

用于页面展示，不直接入库。

| 字段 | 类型 | 说明 |
|---|---|---|
| totalCheckInCount | int | 累计打卡 |
| bestStreak | int | 最佳连续 |
| currentStreak | int | 当前连续 |
| durationDays | int | 持续天数 |

### 6.2 DateStatus

用于月历和热力图。

```text
checked
checkedDuringPause
grace
missed
paused
future
beforeCreated
```

### 6.3 HomeHabitStatus

首页卡片状态。

```text
notChecked
checked
resting
paused
```

## 7. ID 规则

建议使用 UUID。

备份恢复时必须保留备份文件中的原始 ID，不重新生成。

## 8. 日期规则

- 日期使用设备本地日期。
- 业务日期统一保存为 `YYYY-MM-DD`。
- 时间点统一保存为 ISO 8601 字符串。
- MVP 不处理跨时区迁移修正。
