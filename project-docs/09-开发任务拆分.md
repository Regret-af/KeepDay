# 开发任务拆分

## 1. 开发原则

- 先搭建数据和领域规则，再完善页面。
- 先完成核心闭环，再做体验细节。
- 每个写操作都必须有统计重算。
- 宽限期、暂停期、连续天数必须有单元测试。

## 2. V0.1 基础可用版

### 2.1 工程初始化

- 创建 Flutter 项目。
- 配置 Android 包名。
- 接入 Riverpod。
- 接入 go_router。
- 配置主题色和基础组件。
- 配置 lint。

### 2.2 数据库基础

- 接入 Drift。
- 建立 habits 表。
- 建立 check_in_records 表。
- 实现 AppDatabase。
- 实现基础 DAO。
- 实现数据库初始化。

### 2.3 习惯基础能力

- 创建习惯。
- 编辑习惯。
- 软删除习惯。
- 同名检测。
- 表单校验。

### 2.4 今日打卡

- 首页展示今日习惯。
- 今日打卡。
- 取消今日打卡。
- habitId + checkInDate 唯一约束。
- 打卡后更新累计和基础连续缓存。

### 2.5 基础页面

- 首页。
- 创建习惯页。
- 编辑习惯页。
- 基础详情页。
- 设置页占位。

### 3.2 补卡与历史操作

### 3.3 月历与热力图

- 状态图例。

### 3.4 暂停恢复

- 首页暂停折叠区。

## 4. V0.3 提醒版

### 4.1 通知基础

- 接入 flutter_local_notifications。
- 接入 permission_handler。
- 通知权限检查。
- 首页权限提示。
- 设置页权限状态。

### 4.2 提醒配置

- 创建/编辑页提醒开关。
- 提醒时间选择。
- 修改提醒后重排通知。

### 4.3 NotificationLog

- 建立 notification_logs 表。
- 每日通知记录。
- 每 habit 每日最多一次。
- 删除、暂停、恢复后的通知处理。

## 5. V0.4 备份恢复版

### 5.1 JSON 导出

- 定义 BackupPayload。
- 查询导出快照。
- 过滤 deletedAt 不为空的 habit。
- 生成 JSON。
- 调起系统保存/分享。

### 5.2 JSON 恢复

- 文件选择。
- JSON 解析。
- schema 校验。
- 版本校验。
- 覆盖恢复确认。
- 事务式覆盖写入。
- 恢复后统计重算。
- 恢复后通知重建。

## 7. 建议任务顺序

```text
1. 工程初始化
2. Drift 数据库
3. Habit CRUD
4. 首页今日列表
5. 今日打卡
6. 统计计算
7. 详情页
8. 补卡/取消历史打卡
9. 暂停/恢复
10. 月历/热力图
11. 本地通知
12. JSON 导出
13. JSON 恢复
14. 测试补齐
15. 上架候选体验完善
```

## 8. 关键里程碑

### M1：本地数据闭环

验收：

- 可创建习惯。
- 可今日打卡。
- 可关闭重开 App 后数据仍存在。

### M2：规则闭环

验收：

- 宽限期正确。
- 暂停期正确。
- 统计正确。

### M3：提醒闭环

验收：

- 可设置提醒。
- 权限关闭时提示。
- 暂停和删除后不再提醒。

### M4：数据安全闭环

验收：

- 可导出 JSON。
- 可覆盖恢复。
- 恢复失败不破坏当前数据。
