# 本地数据库设计

## 1. 数据库选型

使用 Drift 作为 SQLite 封装。

原因：

- 类型安全。
- 支持事务。
- 支持唯一约束。
- 支持流式查询。
- 测试友好。
- 符合 PRD 中“不裸写 SQL”的要求。

## 3. 软删除策略

Habit 使用软删除。

删除时：

```text
habit.deleted_at = now
habit.updated_at = now
```

不物理删除：

- CheckInRecord。
- HabitPausePeriod。
- NotificationLog。

业务查询默认只查询：

```text
deleted_at IS NULL
```

导出时只导出未删除 Habit 及其关联数据。

## 4. 事务策略

以下操作必须在事务中执行：

- 创建习惯。
- 编辑习惯。
- 删除习惯。
- 暂停习惯。
- 恢复习惯。
- 今日打卡。
- 取消打卡。
- 补卡。
- 覆盖恢复。

写操作标准流程：

```text
transaction
→ 写入或修改业务数据
→ 查询该 habit 所有关联记录
→ 重算统计缓存
→ 更新 habits 缓存字段
→ commit
→ 事务外处理通知计划
```

通知计划不建议放在数据库事务内，因为系统通知 API 不受 SQLite 回滚控制。

## 5. 数据库版本迁移

初始版本：

```text
schemaVersion = 1
```

迁移原则：

- 新增字段提供默认值。
- 删除字段前先兼容至少一个版本。
- 备份恢复 schema 与数据库 schema 分开管理。
- 数据库迁移失败时阻止进入 App，并给出可恢复提示。

## 7. 数据一致性约束

应用层必须校验：

- Habit name 合法。
- gracePerWeek 只能为 0 / 1 / 2。
- reminderEnabled 为 true 时 reminderTime 不为空。
- paused 状态必须存在 endDate 为 null 的暂停区间。
- normal 状态不应存在 endDate 为 null 的暂停区间。
- CheckInRecord 日期不能早于 Habit.createdAt 日期。
- CheckInRecord 日期不能晚于今天。

数据库层必须保证：

- habitId + checkInDate 唯一。
- 主键唯一。
- 必填字段非空。
